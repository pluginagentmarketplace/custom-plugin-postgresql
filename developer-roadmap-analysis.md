## File Structure & Organization

### Repository Structure
```
├── src/
│   ├── data/
│   │   ├── roadmaps/           # Main roadmaps directory (69 folders)
│   │   │   ├── {roadmap-name}/
│   │   │   │   ├── {roadmap-name}.json      # Visual node-based roadmap data
│   │   │   │   ├── {roadmap-name}.md        # Metadata frontmatter
│   │   │   │   ├── content/                 # Individual topic markdown files
│   │   │   │   │   └── {topic}@{id}.md
│   │   │   │   ├── faqs.astro              # FAQ component
│   │   │   │   └── migration-mapping.json   # Version migration data
│   │   ├── best-practices/
│   │   ├── projects/
│   │   ├── question-groups/
│   │   ├── videos/
│   │   └── authors/
│   ├── components/
│   ├── pages/
│   └── lib/
```

### Individual Roadmap Structure
Each roadmap folder contains:

1. **{roadmap-name}.json** - Main roadmap data
   - Visual node-based structure
   - Node types: title, topic, subtopic, paragraph, button, vertical
   - Position data (x, y coordinates)
   - Style properties (colors, fonts, sizing)
   - Legend indicators (purple=recommended, green=alternative)
   - Interactive elements and links

2. **{roadmap-name}.md** - Metadata file with frontmatter
   - Core metadata (title, description, order)
   - Classification tags (role-roadmap, skill-roadmap)
   - Technical specs (dimensions, renderer, URLs)
   - SEO configuration
   - Related roadmaps
   - Publishing information

3. **content/** directory
   - Individual markdown files for each topic
   - Naming: `{topic-name}@{unique-id}.md`
   - Detailed explanations, resources, examples
   - Can contain 100+ topic files per roadmap

4. **faqs.astro** (optional)
   - Frequently asked questions component

5. **migration-mapping.json** (optional)
   - Version migration data

## Data Structure & Format

### JSON Roadmap Structure
```json
{
  "nodes": [
    {
      "id": "unique-identifier",
      "type": "topic|subtopic|title|paragraph|button|vertical",
      "position": { "x": 0, "y": 0 },
      "data": {
        "label": "Display text",
        "style": {
          "fontSize": "16px",
          "backgroundColor": "#color",
          "borderColor": "#color",
          "textAlign": "center"
        },
        "legend": "recommended|alternative",
        "href": "/path/to/resource"
      },
      "measured": { "width": 100, "height": 50 },
      "zIndex": 1
    }
  ]
}
```

### Markdown Frontmatter Structure
```yaml
---
order: 1-100                    # Display order/priority
briefTitle: "Short Name"        # Condensed title
title: "Full Title"             # Complete title
briefDescription: "..."         # Short description
description: "..."              # Full description

# Technical Configuration
renderer: editor                # Rendering engine (balsamiq|editor)
jsonUrl: /jsons/roadmaps/{name}.json
pdfUrl: /pdfs/roadmaps/{name}.pdf
hasTopics: true
isForkable: false
isNew: false
isHidden: false
isUpcoming: false

# Classification
tags:
  - roadmap
  - main-sitemap
  - role-roadmap              # OR skill-roadmap

# Relationships
relatedRoadmaps:
  - roadmap-id-1
  - roadmap-id-2
relatedQuestions:
  - question-group-id

# Dimensions
dimensions:
  width: 968
  height: 2000

# SEO
seo:
  title: "SEO optimized title"
  description: "SEO description"
  keywords:
    - keyword1
    - keyword2

# Schema
schema:
  headline: "Schema headline"
  datePublished: "2023-01-05"
  dateModified: "2024-10-03"
  imageUrl: "https://roadmap.sh/roadmaps/{name}.png"

# Sitemap
sitemap:
  priority: 1
  changefreq: monthly

# Optional
partner:
  description: "Partner info"
  link: "https://partner.com"
  linkText: "Learn more"
---
```
